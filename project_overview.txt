안녕하세요. 저는 LG전자의 IT아웃소싱팀에서 모니터의 공급을 담당하고 있습니다.
LG전자에서 생산되는 모니터 중 일부는 해외의 아웃소싱 업체에서 생산하여 공급되는 것들도 있습니다.
아웃소싱 비즈니스에는 다양한 업무의 영역이 있습니다만, 그 중에서 수많은 모델과 여러 국가로 선적되는 모니터 공급을 안정적으로 관리하는 업무도 쉽지는 않습니다.
이러한 복잡한 업무를 보다 효과적으로 관리하기 위하여 각 아웃소싱 업체에서 생산 후 선적되는 정보를 등록하고 현황을 조회할 수 있는 시스템을 웹으로 개발하고자 합니다. 
이렇게하면 모든 정보가 웹에 축적되어 언제나 누구든지 웹에 접속하여 최신의 정보를 조회하고 각 유저가 자신이 관련된 데이터의 추가, 수정, 삭제 등을 하여 보다 효율적으로 선적 정보를 파악하며
개인적으로 해야할 문서 작업을 획기적으로 줄일 수 있을 것 입니다.

 * 웹서비스의 명칭 : 로지스피어(Logi-Sphere) 
 * 로지스피어의 뜻 : Logistics(물류) + Sphere(구, 세계)로, 글로벌 물류 네트워크를 관리하는 느낌을 줍니다.
 * 웹서비스에서 구현해야 할 주요 기능
    1) Main Page
        "로지스피어" 시스템의 Nain Page에는 로지스피어 시스템의 이름과 이름에 걸맞는 이미지와 Logo가 배치되어 있으며, 시스템을 이용할 유저의 Log-In 과 회원 가입 버튼이 있어야 함
    2) 회원가입 페이지
        - Main Page에서 회원 가입 버튼을 누르면 회원가입 페이지로 이동하며, 회원가입 페이지에는 이름, 소속(업체명), 담당업무, ID, Password, 이메일주소, 캐릭터이미지를 선택하여 해당 정보를 LG전자의 시스템 관리자에게 가입 요청을 함
        - 회원 가입 시 ID 중복은 없도록 해야 하고, password도 최소 네글자 이상으로 등록되어야 함
        - 회원으로 등록할 수 있는 유저의 업체는 크게 LG전자, 아웃소싱업체, 물류업체라는 3개의 카테고리로 나뉨
        - 회원가입 form이 작성되어 요청되면 LG전자 시스템 관리자가 시스템에 login 했을 때 User 관리 페이지에 신청자 정보가 나타나게 되며 각 필드를 확인 후 이상없으면 승인하면 다음부터 신청자는 신청한 ID/Password 정보로 login 할수 있게 됨
        - 신청자의 신청 정보 중 업체명은 시스템관리자가 확인한 뒤, 별도의 통일된 업체명으로 부여하여 저장되어야 함.
    3) "로지스피어"에서 구현해야 할 각 업체별 담당자의 주요 업무 내용
        - LG전자의 유저 : 매주 각 아웃소싱 업체로 확정된 선적 계획을 업로드하고 이것이 DB로 저장이 되며, 저장된 DB의 조회/수정/삭제가 가능해야 함
        - 각 아웃소싱업체의 유저 : 자신의 업체에 할당된 선적 계획만 조회가 가능해야 하며 조회된 각 개별의 선적리스트 중 부킹해야 할 것을 선택한 뒤 필요한 부킹 요청 정보를 입력하고 물류 업체의 담당자를 선택하여 선적 부킹 요청을 함
                                (보안상의 이유로 자신의 업체 외에 다른 업체의 선적 관련 정보 등이 누설되지 않아야 함)
        - 물류업체의 유저 : 각 아웃소싱업체의 담당자가 작성한 선적 부킹 요청을 My Booking Request 화면에서 확인하고 확정된 선적 정보를 입력/수정을 할 수 있어야 함
    4) 각 유저가 로그인 후 나타나는 페이지
        - LG전자의 유저
            a. Shipment Dashboard : 업체별/기간별 선적 계획 조회 및 선적 계획 대비 선적 부킹율/선적 확보율/컨테이너 수량/40HC사용율 조회 (그래프와 표로 시각화)
            b. Booking Detail : 조회 조건에 따른 세부 선적 정보 결과 조회
            c. Upload Shipment Plan : 신규 선적 계획을 엑셀 파일로 업로드하여 DB에 추가하는 화면
        - 각 아웃소싱업체의 유저
            a. My Booking : 자신에 업체에 할당된 선적 계획을 조회하고 선적 부킹을 요청하고자 하는 선적계획을 선택하여 Start Booking을 누르면 화면 하단에 세부적인 부킹 요청 정보를 입력하고 제출 버튼을 눌러 요청하는 화면
            b. Shipment Dashboard : 기간별 선적 계획 및 선적 계획 대비 선적 부킹율/선적 확보율/컨테이너 수량/40HC사용율 등 조회 (그래프와 표로 시각화)
            c. Booking Detail : 조회 조건에 따른 세부 선적 정보 결과 조회
        - 물류업체 유저
            a. My booking request : 자신에게 요청된 부킹 요청을 조회 후, 선적 정보를 입력하고 저장함. 중요한 점은 최초에 입력된 정보에서 입력 필드의 어떤 항목이라도 변경이 발생하면 변경 사유도 입력받아 최초 입력 정보 대비 어떻게 변동되었는지도 추적할 수 있게 되어야 함.
            b. Shipment Dashboard : 기간별 선적 계획 및 선적 계획 대비 선적 부킹율/선적 확보율/컨테이너 수량/40HC사용율 등 조회 (그래프와 표로 시각화)
            c. Booking Detail : 조회 조건에 따른 세부 선적 정보 결과 조회
    
    5) 시스템 관리자만 볼 수 있는 User 관리 페이지
        - LG전자의 시스템 관리자는 신규 회원 가입 요청을 검토 후 회원 등록을 승인 또는 경우에 따라 회원가입된 유저가 더 이상 Log-in 하지 못하도록 삭제할 수 있어야 함
        - 회원가입일 기준으로 3개월 이상 log-in을 하지 않는 유저는 자동으로 유저의 Login 정보가 삭제되도록 해야 함
        - 유저가 Log-in 할 때 마다 그 유저의 로그인/로그아웃 시간 및 데이터를 입력/조회/수정 등의 작업을 할 때 어떤 작업을 했는지 작업의 이름 및 그 시간을 별도의 DB에 저장되어 시스템 관리자가 조회할 수 있어야 함
        - 유저가 Log-in 후 별도로 로그아웃을 누르지 않으면 1시간 후 자동으로 Log-out 이 되도록 해야 함
    6) 회원 정보 Page : 각 유저는 자신의 가입 정보의 조회, 캐릭터의 변경 등을 할 수 있음
    7) 공지사항 게시판 : 시스템 관리자가 주요 공지사항을 등록하는 게시판
    8) 자유 게시판 : 회원들이 자유롭게 글을 적거나 수정 또는 삭제하고 각 글에 대해서 또 다른 유저가 답글을 달 수 있는 게시판
    9) Fun Time : 간단한 캐주얼 게임을 하고 게임의 점수로 각 유저의 Ranking을 표시하는 페이지

회원가입 페이지에서 선택될 등록업체 리스트
 1) LG전자 : LGE
 2) 아웃소싱업체 : AUO, TPV, BOVT, MOKA, KTC, etc
 3) 물류업체 정보 : Pantos, etc

LG전자의 선적계획에 포함되는 정보 : From Site, To Site, Country, Final Destination, Mapping Model.Suffix, 선적수량, PO Number, 
선적 부킹을 위해 필요한 정보 : Shipper, Consignee, Notify, 도착지 정보(Final Destination), Container 종류, Cargo ready date, 운송모드(Sea, Air, Train, Truck) 등
물류담당자가 업데이트해야할 선적 정보 : ETD/ETA/Vessel Name/Voyage/SO/SI Cut Time/CY CLS TIME/BL Number
